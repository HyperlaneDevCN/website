如何使用 Hyperlane 浏览器

Hyperlane 浏览器可用于查找任何已部署 Hyperlane 的网络上的消息。深入了解消息以获取有关其属性和状态的更多详细信息。

来自/发往核心链的消息可以通过交易哈希、消息ID或发送者/接收者地址进行搜索。对于其他链，可以通过提供链配置来配置浏览器以搜索消息。

搜索消息​

Hyperlane 浏览器支持使用以下任何值搜索消息：

  • 发起消息的交易哈希（源交易）
  • 传递消息的交易哈希（目标交易）
  • 发送源交易的账户地址
  • 发送目标交易的账户地址
  • 消息发送者的地址
  • 消息接收者的地址
  • 消息的ID

要搜索，请在顶部搜索栏中输入您的查询。您可以使用源链、目标链和时间过滤器选项来缩小搜索范围。

信息

注意，默认情况下，浏览器只会在核心 Hyperlane 链上查找消息。

要查看发送到和/或来自其他链的消息，请参阅配置 PI 链。

无许可互操作性​

浏览器可以配置为搜索任何已部署 Hyperlane 的链上的消息。有关详细信息，请参阅配置 PI 链。

调试消息​

如果您正在尝试调试消息传递问题，浏览器调试页面有一些有用的提示。
